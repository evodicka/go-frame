# Style and Conventions

## Backend (Go)
- Standard Go naming conventions (CamelCase).
- Clean separation of concerns (model, persistence, api, admin-api).
- Dependency injection for storage implementation.
- Use of `init()` for global loggers (Info, Warning, Error).
- Comprehensive test coverage using `go test`.

## Frontend (Angular)
- Standard Angular 13 project structure.
- TypeScript for type safety.
- Separation of components, templates, and styles.
